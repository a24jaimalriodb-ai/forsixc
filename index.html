📚 Pràctica 1: Configuració del Servidor Web Apache

🎯 Objectiu de la Pràctica

L'empresa Advocats Pedralbes S.L. necessita instal·lar un servidor WEB per albergar diferents llocs web. Aquesta pràctica cobreix la configuració completa d'Apache amb Virtual Hosts.

Tasca 1: Configuració del Hostname

Configurar el hostname de la màquina amb el vostre nom i primera lletra del cognom.

sudo hostnamectl set-hostname gabrielm
hostname

Resultat: El sistema mostrarà el nou hostname "gabrielm"

Tasca 2: Actualització del Sistema

Realitzar una actualització del sistema per assegurar que tots els paquets estiguin a la última versió.

sudo apt update

Aquest comandament actualitzarà la llista de paquets disponibles i les seves versions.

Tasca 3: Instal·lació d'Apache2

Instal·lar el servidor web Apache2 i verificar la instal·lació.

sudo apt install apache2
dpkg -s apache2

Verificació: El paquet ha de mostrar "Status: install ok installed"

Tasca 4: Configuració de la Interfície de Xarxa

Configurar l'adaptador de xarxa enp2s0 amb els següents paràmetres:

🌐 Adreça IP:
192.168.200.1
🔢 Màscara de subxarxa:
255.255.255.0
🚪 Porta d'enllaç:
192.168.200.1
🔍 DNS:
8.8.8.8
ip a

Aquest comandament mostrarà la configuració de totes les interfícies de xarxa.

Tasca 5: Configuració del Virtual Host

Crear el fitxer de configuració per al domini www.advocatspedralbes.cat

<VirtualHost *:80> ServerName advocatspedralbes.cat ServerAlias www.advocatspedralbes.cat DocumentRoot /var/www/advocatspedralbes <Directory /var/www/advocatspedralbes> DirectoryIndex index.html Options Indexes </Directory> </VirtualHost>

Activació del lloc web:

sudo /usr/sbin/a2ensite advocatspedralbes.cat.conf
sudo systemctl reload apache2

Tasca 6: Verificació del Servei

Comprovar que el servei Apache està actiu i funcionant correctament.

sudo systemctl status apache2

Estat esperat: Active (running) en verd

Provar des del navegador: http://localhost o http://127.0.0.1

Tasca 7: Creació del Directori del Lloc Web

Crear el directori per al lloc web i el fitxer index.html

sudo mkdir -p /var/www/advocatspedralbes
sudo nano /var/www/advocatspedralbes/index.html

El fitxer index.html ha de contenir una pàgina web simple amb informació sobre "Advocats Pedralbes" i el vostre nom.

Tasca 8: Prova des del Client Windows

Configurar un client Windows amb els següents paràmetres de xarxa:

🌐 IP:
192.168.200.81
🔢 Màscara:
255.255.255.0
🚪 Porta d'enllaç:
192.168.200.1
🔍 DNS:
8.8.8.8

Configuració del fitxer hosts a Windows:

Ruta: C:\Windows\System32\drivers\etc\hosts

192.168.200.1 www.advocatspedralbes.cat

Obrir el navegador i accedir a: www.advocatspedralbes.cat

Tasca 9: Configuració del Segon Virtual Host

Configurar el domini www.metgespedralbes.cat seguint els mateixos passos que per al primer domini.

📋 Passos a seguir:
  1. Crear el fitxer de configuració: /etc/apache2/sites-available/metgespedralbes.cat.conf
  2. Crear el directori: /var/www/metgespedralbes
  3. Crear el fitxer index.html amb contingut personalitzat
  4. Activar el lloc: sudo a2ensite metgespedralbes.cat.conf
  5. Reiniciar Apache: sudo systemctl reload apache2

Prova des del Client Linux:

🔐 Pràctica 2: Autenticació Bàsica Apache

🎯 Objectiu de la Pràctica

Implementar autenticació bàsica a Apache utilitzant htpasswd per protegir l'accés als llocs web configurats anteriorment.

Tasca 1: Configuració de l'Autenticació per advocatspedralbes.cat

Afegir la configuració d'autenticació al fitxer de Virtual Host:

<Directory /var/www/advocatspedralbes> AuthType Basic AuthName "Control d'accés" AuthUserFile /etc/apache2/usuaris.txt Require valid-user </Directory>

Tasca 2: Creació d'Usuaris amb htpasswd

Crear el fitxer d'usuaris i afegir els usuaris autoritzats:

Primer usuari:

htpasswd -c /etc/apache2/usuaris.txt gabrielmormontoy

Us demanarà la contrasenya: smix

Segon usuari:

htpasswd /etc/apache2/usuaris.txt gabrielmormontoy2

Important: No utilitzar el paràmetre -c per al segon usuari, ja que esborrarà el primer!

💡 Per què la contrasenya es veu diferent?

Les contrasenyes es guarden xifrades (hash) al fitxer usuaris.txt per seguretat. Això evita que qualsevol persona que accedeixi al fitxer pugui veure les contrasenyes reals.

Tasca 3: Verificació del Servei

sudo systemctl status apache2

El servei ha d'estar "Active (running)"

Tasca 4: Prova des del Client Windows

Accedir a www.advocatspedralbes.cat des d'un navegador. El sistema demanarà:

🔑 Credencials d'accés:
  • Nom d'usuari: gabrielmormontoy
  • Contrasenya: smix

Tasca 5: Configuració de l'Autenticació per metgespedralbes.cat

Repetir el mateix procés per al segon domini:

<Directory /var/www/metgespedralbes> AuthType Basic AuthName "Control d'accés" AuthUserFile /etc/apache2/usuaris.txt Require valid-user </Directory>

Tasca 6: Creació d'Usuaris per metgespedralbes.cat

Utilitzar els mateixos usuaris o crear-ne de nous utilitzant htpasswd.

htpasswd /etc/apache2/usuaris.txt nomUsuari
sudo systemctl reload apache2

Tasca 7: Prova des del Client Linux

Configurar el client Linux i editar /etc/hosts:

127.0.0.1 localhost 127.0.1.1 debian 192.168.200.1 www.advocatspedralbes.cat 192.168.200.1 www.metgespedralbes.cat

Accedir a www.metgespedralbes.cat i introduir les credencials quan es sol·licitin.

⚠️ Punts Importants de Seguretat

  • Les contrasenyes es xifren automàticament amb htpasswd
  • El fitxer usuaris.txt ha de tenir permisos restrictius
  • L'autenticació bàsica envia credencials en text pla (usar HTTPS en producció)
  • Reiniciar Apache després de cada canvi de configuració

Comandaments Principals Utilitzats

📝 Resum de Comandaments:
  • htpasswd -c fitxer usuari - Crear fitxer nou amb primer usuari
  • htpasswd fitxer usuari - Afegir usuari a fitxer existent
  • sudo systemctl reload apache2 - Recarregar configuració Apache
  • sudo systemctl status apache2 - Verificar estat del servei
  • sudo cat /etc/apache2/usuaris.txt - Veure usuaris creats

🔐 Práctica 2: Autenticación Básica Apache

Tarea 1: Configuración de la Autenticación para advocatspedralbes.cat

Añadir la configuración de autenticación al archivo de Virtual Host:

<Directory /var/www/advocatspedralbes> AuthType Basic AuthName "Control de acceso" AuthUserFile /etc/apache2/usuaris.txt Require valid-user </Directory>

Crear el archivo de usuarios y añadir los usuarios autorizados:

Primer usuario:

htpasswd -c /etc/apache2/usuaris.txt gabrielmormontoy

Las contraseñas se guardan cifradas (hash) en el archivo usuaris.txt por seguridad. Esto evita que cualquier persona que acceda al archivo pueda ver las contraseñas reales.

Tarea 3: Verificación del Servicio

sudo systemctl status apache2

El servicio debe estar "Active (running)"

Tarea 4: Prueba desde el Cliente Windows

Acceder a www.advocatspedralbes.cat desde un navegador. El sistema pedirá:

🔐 实践2:Apache基本身份验证

🎯 实践目标

使用htpasswd在Apache中实现基本身份验证,以保护对先前配置的网站的访问。

任务1:为advocatspedralbes.cat配置身份验证

在虚拟主机文件中添加身份验证配置:

<Directory /var/www/advocatspedralbes> AuthType Basic AuthName "访问控制" AuthUserFile /etc/apache2/usuaris.txt Require valid-user </Directory>

任务2:使用htpasswd创建用户

创建用户文件并添加授权用户:

第一个用户:

htpasswd -c /etc/apache2/usuaris.txt gabrielmormontoy

将要求输入密码: smix

第二个用户:

htpasswd /etc/apache2/usuaris.txt gabrielmormontoy2

重要: 不要为第二个用户使用-c参数,因为它会删除第一个用户!

💡 为什么密码看起来不同?

密码在usuaris.txt文件中以加密(哈希)形式保存以确保安全。这可以防止访问文件的任何人看到真实密码。

任务3:服务验证

sudo systemctl status apache2

服务应该是 "Active (running)"

任务4:从Windows客户端测试

从浏览器访问www.advocatspedralbes.cat。系统将要求:

🔑 访问凭据:
  • 用户名: gabrielmormontoy
  • 密码: smix

任务5:为metgespedralbes.cat配置身份验证

对第二个域名重复相同的过程:

<Directory /var/www/metgespedralbes> AuthType Basic AuthName "访问控制" AuthUserFile /etc/apache2/usuaris.txt Require valid-user </Directory>

任务6:为metgespedralbes.cat创建用户

使用相同的用户或使用htpasswd创建新用户。

htpasswd /etc/apache2/usuaris.txt 用户名
sudo systemctl reload apache2

任务7:从Linux客户端测试

配置Linux客户端并编辑/etc/hosts:

127.0.0.1 localhost 127.0.1.1 debian 192.168.200.1 www.advocatspedralbes.cat 192.168.200.1 www.metgespedralbes.cat

访问www.metgespedralbes.cat并在请求时输入凭据。

⚠️ 重要安全要点

  • 密码会自动使用htpasswd加密
  • usuaris.txt文件必须有限制性权限
  • 基本身份验证以明文形式发送凭据(生产环境使用HTTPS)
  • 每次配置更改后重启Apache

主要使用的命令

📝 命令摘要:
  • htpasswd -c 文件 用户 - 创建新文件并添加第一个用户
  • htpasswd 文件 用户 - 向现有文件添加用户
  • sudo systemctl reload apache2 - 重新加载Apache配置
  • sudo systemctl status apache2 - 验证服务状态
  • sudo cat /etc/apache2/usuaris.txt - 查看创建的用户

Görev 9: İkinci Sanal Host Yapılandırması

İlk domain için aynı adımları izleyerek www.metgespedralbes.cat domain'ini yapılandırın.

📋 Takip edilecek adımlar:
  1. Yapılandırma dosyası oluşturun: /etc/apache2/sites-available/metgespedralbes.cat.conf
  2. Dizin oluşturun: /var/www/metgespedralbes
  3. Özel içerikli index.html dosyası oluşturun
  4. Siteyi etkinleştirin: sudo a2ensite metgespedralbes.cat.conf
  5. Apache'yi yeniden başlatın: sudo systemctl reload apache2

🔐 Uygulama 2: Apache Temel Kimlik Doğrulama

🎯 Uygulamanın Amacı

Daha önce yapılandırılmış web sitelerine erişimi korumak için htpasswd kullanarak Apache'de temel kimlik doğrulamayı uygulamak.

Görev 1: advocatspedralbes.cat için Kimlik Doğrulama Yapılandırması

Sanal Host dosyasına kimlik doğrulama yapılandırması ekleyin:

<Directory /var/www/advocatspedralbes> AuthType Basic AuthName "Erişim Kontrolü" AuthUserFile /etc/apache2/usuaris.txt Require valid-user </Directory>

Görev 2: htpasswd ile Kullanıcı Oluşturma

Kullanıcı dosyası oluşturun ve yetkili kullanıcılar ekleyin:

İlk kullanıcı:

htpasswd -c /etc/apache2/usuaris.txt gabrielmormontoy

Şifre isteyecek: smix

İkinci kullanıcı:

htpasswd /etc/apache2/usuaris.txt gabrielmormontoy2

Önemli: İkinci kullanıcı için -c parametresini kullanmayın, çünkü birincisini silecektir!

💡 Şifre neden farklı görünüyor?

Şifreler güvenlik için usuaris.txt dosyasında şifrelenmiş (hash) olarak saklanır. Bu, dosyaya erişen herhangi birinin gerçek şifreleri görmesini engeller.

Görev 3: Servis Doğrulama

sudo systemctl status apache2

Servis "Active (running)" olmalıdır

Görev 4: Windows İstemcisinden Test

  • Şifre: smix

Görev 5: metgespedralbes.cat için Kimlik Doğrulama Yapılandırması

İkinci domain için aynı işlemi tekrarlayın:

<Directory /var/www/metgespedralbes> AuthType Basic AuthName "Erişim Kontrolü" AuthUserFile /etc/apache2/usuaris.txt Require valid-user </Directory>

Görev 6: metgespedralbes.cat için Kullanıcı Oluşturma

Aynı kullanıcıları kullanın veya htpasswd kullanarak yenilerini oluşturun.

htpasswd /etc/apache2/usuaris.txt kullaniciAdi

⚠️ Önemli Güvenlik Noktaları

  • Şifreler htpasswd ile otomatik olarak şifrelenir
  • usuaris.txt dosyası kısıtlayıcı izinlere sahip olmalıdır
  • Temel kimlik doğrulama kimlik bilgilerini düz metin olarak gönderir (üretimde HTTPS kullanın)
  • Her yapılandırma değişikliğinden sonra Apache'yi yeniden başlatın
